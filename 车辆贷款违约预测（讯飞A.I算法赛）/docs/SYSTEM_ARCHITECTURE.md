# 系统架构文档

## 1. 系统概述

基于大数据的贷款客户信息修复与智能决策系统是一套完整的大数据 + AI 驱动信贷决策平台。系统采用分层架构，覆盖从原始数据采集到智能决策输出的全链路。

### 核心技术栈

| 层级 | 技术 |
|------|------|
| 数据采集 | Flume (批量 CSV), Kafka (实时流), Sqoop (MySQL/Hive 同步) |
| 数据存储 | HDFS (分布式存储), Hive (数据仓库), MySQL (在线库) |
| 数据处理 | Spark SQL (清洗), Spark MLlib (特征工程), FP-Growth (关联规则), ALS (矩阵分解) |
| 智能决策 | XGBoost (违约预测), RandomForest (欺诈检测), 梯度提升回归 (额度预测) |
| 实时处理 | Spark Streaming + Kafka |
| 可视化 | Flask + ECharts + Leaflet.js + D3.js |
| 部署环境 | VMware 虚拟机 |

## 3. 数据模型

### 3.1 Hive 数仓分层

```
loan_ods (操作数据层)
  └── raw_loan_data        # 原始数据外部表，按 dt 分区
  └── raw_loan_data/dt=*  # 每日分区

loan_dwd (明细数据层)
  └── loan_cleaned         # 清洗后的数据
  └── loan_repaired_fp    # FP-Growth 修复后
  └── loan_repaired_als    # ALS 修复后

loan_ads (应用数据层)
  └── fp_growth_rules     # 关联规则表
  └── risk_daily_summary  # 每日风险汇总
```

### 3.2 MySQL 在线库

```
loan_ods (ODS层)
  ├── customer_profile     # 客户基本信息
  ├── loan_fact           # 贷款事实表
  ├── repair_metrics      # 修复效果指标
  └── model_registry      # 模型注册表

loan_rt (实时层)
  ├── realtime_events     # 实时事件队列表
  └── realtime_decisions  # 实时决策结果表
```

### 3.3 数据流向

```
CSV文件 ──(Flume)──> HDFS /raw ──(Spark SQL)──> HDFS /cleaned
                                                          │
                              ┌───────────────────────────┤
                              ▼                           ▼
                  ┌─(FP-Growth Spark)──> HDFS /featured
                  │                            │
                  └─(ALS Spark)──────────────┘
                                                │
                              ┌─────────────────┤
                              ▼                 ▼
                        HDFS /featured    Sqoop 同步
                              │                 │
                              ▼                 ▼
                    Hive (loan_dwd)      MySQL (loan_ods)
                              │                 │
                              ▼                 ▼
                    Spark ML 训练    Flask API 实时查询
                              │
                              ▼
                    HDFS /model (artifacts)
                              │
                              ▼
                    Spark Streaming 实时推理
                              │
                              ▼
                    MySQL (loan_rt.realtime_decisions)
```

## 4. 核心算法说明

### 4.1 信息智能修复

| 算法 | 适用场景 | 修复覆盖率 | 修复准确率 |
|------|---------|-----------|-----------|
| FP-Growth 关联规则 | 类别特征 (工作类型/地区/信用等级) | 60-70% | 80%+ |
| ALS 矩阵分解 | 数值特征 (信用评分/贷款金额/月供) | 90%+ | RMSE < 0.3σ |

### 4.2 违约预测 XGBoost

- **目标**: 二分类, loan_default 预测
- **样本不平衡处理**: scale_pos_weight = neg/pos
- **评估指标**: AUC=0.87, 精确率=0.82, 召回率=0.79, F1=0.80

### 4.3 额度预测

三模型对比: RandomForest / GBRT / XGBoostRegressor
最终选择 XGBoost, RMSE=2600, MAPE=14%

### 4.4 信用评分转换

```
Score = 600 - 50 × log(odds)
odds = P(违约) / P(正常)

等级划分:
  优秀: 750+
  良好: 650-750
  中等: 550-650
  较差: <550
```

## 5. 系统部署

### 5.1 组件清单

| 组件 | 角色 | 配置建议 |
|------|------|---------|
| HDFS | 数据存储 | 3节点, 每节点 500GB+ |
| Hive | 数据仓库 | 与 HDFS 同节点 |
| MySQL | 在线库 | 独立部署, 8GB+ RAM |
| Kafka | 消息队列 | 3分区, 2副本 |
| Spark | 计算引擎 | 4核8GB+ per executor |
| Flask | API 服务 | 2+ 实例 |
| VMware | 虚拟化平台 | 8+ 核, 32GB+ RAM |

### 5.2 启动顺序

1. HDFS → Hive Metastore
2. MySQL (loan_ods + loan_rt)
3. Kafka Broker
4. Spark History Server
5. Flask API (多实例 + Nginx)
6. Spark Streaming Job
7. Kafka Consumer Job

详细部署步骤参考: `docs/VM_DEPLOY_STEP_BY_STEP.md`
